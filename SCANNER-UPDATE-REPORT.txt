FirePhage Security Scanner Update
Date: 2026-03-07

Summary
- Reworked the scanner around an integrity-first model based on the malware-plugin review.
- Repository checks now happen before heuristic malware scoring.
- The admin UI now exposes richer scan state and finding metadata.

Scanner changes
- WordPress core files are verified against official core checksums.
- WordPress.org plugin files are verified against official plugin package checksums.
- WordPress.org theme files are verified against official theme package checksums.
- Custom files outside official package inventories are added to a local clean baseline.
- A monitor cron can restart a stalled background scan if batch execution stops progressing.
- Heuristic findings are now separated from integrity mismatches.

UI changes
- Scan copy now explains the checksum-plus-baseline model instead of a simple extension list.
- Progress text now reports trusted files, integrity mismatches, and suspicious findings.
- Latest findings now show type, confidence, and source badges.
- The start button remains disabled while a scan is active.

Housekeeping
- Plugin deactivation now clears the scan monitor cron hook.
- Plugin uninstall now removes the local scan baseline option.
